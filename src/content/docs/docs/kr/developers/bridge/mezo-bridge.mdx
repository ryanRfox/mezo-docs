Mezo 브리지는 Bitcoin, Ethereum, Mezo 간의 네이티브 자산 이동을 가능하게 합니다. 아래는 설계 RFC를 참조한 주요 흐름입니다.

### Mezo로 브리지 인

Bitcoin → Mezo

<Steps>
1. Mezo 수신자를 식별하는 메타데이터와 함께 브리지 제어 주소로 BTC를 잠급니다.

2. Mezo 브리지 컨트랙트에 증명과 인증을 제출합니다.

3. 컨트랙트가 증명을 검증합니다.

4. Mezo에서 수신자에게 해당 자산을 발행합니다.
</Steps>

Ethereum → Mezo

<Steps>
1. Mezo 브리지 컨트랙트를 통해 Ethereum에서 ERC-20을 잠그고, Mezo 수신자와 함께 `AssetLocked` 이벤트를 발생시킵니다.

2. 브리지 밸리데이터가 해당 이벤트를 수신합니다.

3. 브리지 밸리데이터가 이벤트 포함 및 밸리데이터 쿼럼을 검증합니다.

4. Mezo에서 수신자에게 해당 자산을 발행합니다.
</Steps>

### Mezo에서 브리지 아웃

Mezo → Bitcoin

<Steps>
1. Bitcoin 수신자 주소를 지정하여 Mezo에서 자산을 소각합니다.

2. Mezo 브리지 컨트랙트에 증명과 인증을 제출합니다.

3. 컨트랙트가 증명을 검증합니다.

4. 검증이 완료되면 수신자 주소로 BTC가 릴리스됩니다.
</Steps>

Mezo → Ethereum

<Steps>
1. Ethereum 수신자를 지정하여 Mezo에서 자산을 소각합니다.

2. 브리지 밸리데이터가 `AssetsUnlocked` 이벤트를 수신합니다.

3. 브리지 밸리데이터가 이벤트 포함 및 밸리데이터 쿼럼을 검증합니다.

4. 검증이 완료되면 수신자에게 ERC-20이 릴리스됩니다.
</Steps>

### 아키텍처

- **소스에서 잠금/소각**: 자산은 Bitcoin/Ethereum에서 잠기거나 (또는 Mezo에서 소각되며) 대상 수신자와 타겟 체인을 식별하는 데이터와 함께 처리됩니다.
- **인증 및 릴레이**: 오프체인 클라이언트가 소스 체인 이벤트에 대한 인증을 생성하고 증명을 목적지 체인으로 릴레이합니다.
- **목적지에서 발행/릴리스**: 유효한 인증 및 증명 검증이 완료되면, 목적지 컨트랙트가 래핑된 자산을 발행하거나 잠긴 자금을 릴리스합니다.

### 핵심 흐름

<Steps>
1. 사용자가 목적지 주소와 금액을 지정하여 브리지 전송을 시작합니다.

2. 잠금/소각 후 소스 체인 컨트랙트/이벤트가 발생하며, 이벤트에는 고유한 시퀀스 ID와 매개변수가 포함됩니다.

3. 오프체인 클라이언트가 RFC 임계값에 따라 확인/최종성을 수집하고 증명+인증을 목적지 컨트랙트에 제출합니다.

4. 목적지에서 검증:
   - 이벤트 진위성
   - 밸리데이터 쿼럼/서명

5. 성공 시, 목적지에서 발행/릴리스를 실행하고 완료를 기록합니다.
</Steps>
